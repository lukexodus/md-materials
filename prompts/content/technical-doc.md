Write **{{doc_type}}** documentation for: **{{subject}}**

**Audience:** {{audience}}

Documentation requirements:
- Lead with what it is and what problem it solves (1 short paragraph).
- Structure with clear headers.
- Include a quickstart or minimal working example early.
- Cover: installation/setup, usage, configuration options, common errors.
- Use consistent terminology throughout.
- Do not document features that don't exist yet.

Tone: precise and direct. Avoid marketing language.
