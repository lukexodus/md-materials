Write a technical blog post on: **{{topic}}**

**Target audience:** {{audience}}
**Angle / thesis:** {{angle}}

Structure:
1. **Hook** — Open with something specific: a problem, a surprising fact, or a concrete scenario. No generic intros.
2. **Body** — Develop the angle with substance. Use examples, code snippets, or data where relevant.
3. **Conclusion** — What should the reader walk away knowing or doing differently?

Constraints:
- Do not use the phrase "In conclusion" or "In this post".
- Do not pad. Every paragraph should add something.
- Use subheadings if the post is long enough to need navigation.
